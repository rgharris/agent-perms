# Project Naming Candidates

Shortlist of replacement names for `agent-perms`, vetted against GitHub and the web for conflicts.

## Top picks

| Name         | Meaning                     | Notes                                            |
| ------------ | --------------------------- | ------------------------------------------------ |
| **semgate**  | Semantic gate               | Concise, memorable, zero conflicts found         |
| **licet**    | Latin for "it is permitted" | Elegant, unique, fits the domain perfectly       |
| **tiergate** | Tier-based gating           | Descriptive, instantly communicates what it does |

## Also available

| Name      | Meaning                       | Notes                                                 |
| --------- | ----------------------------- | ----------------------------------------------------- |
| execguard | Guards execution by tier      | Clear but slightly generic                            |
| semexec   | Semantic execution control    | Pairs well with semgrep-style branding                |
| fencepost | Defines permission boundaries | "Fencepost error" is a CS term but not a product name |
| cmdtier   | Command + tier                | Plain, searchable, self-documenting                   |
| semperms  | Semantic permissions          | Echoes Latin _semper_ (always)                        |
